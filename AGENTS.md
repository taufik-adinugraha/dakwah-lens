# Dakwah-Lens — agent rules

Read the PRD (`Dakwah-Lens_PRD_v0.4.pdf`) for full product context. Owner: Sukses & Berkah Group · Author: Taufik Adi.

## Monorepo layout

- `web/` — Next.js 16 frontend. See `web/AGENTS.md` for Next.js-specific rules.
- `api/` — FastAPI + Celery backend. See `api/` files; uv-managed Python 3.12 in `src/api/`.
- `docker-compose.yml` — local Postgres, Qdrant, Redis.

## Locked decisions (current as of 2026-05-25)

| Decision | Value |
|---|---|
| Languages | Indonesian (primary) + English via next-intl |
| Embedding model | `text-embedding-3-large` (OpenAI) |
| Sentiment (all platforms) | Gemini Flash-Lite (`services/sentiment.py`) — single classifier across mainstream + social since 2026-05-25; IndoBERT removed after a manual eval showed ~14% accuracy on X positives |
| Da'wah classifier / opportunity / topic discovery / rerank | Gemini 2.5 Flash-Lite |
| Brief synthesis | Gemini 2.5 Pro (fallback: Claude Sonnet 4.6) |
| Vector store | Self-hosted Qdrant in docker compose |
| Hosting | IDCloudHost VPS (Indonesia residency per UU PDP §27/2022) |
| Auth | NextAuth.js (email + Google OAuth) |
| Multi-tenancy | `organizations` + `org_members` (owner/admin/member), app-level scoping |
| Kitab corpus v0 | Qur'an (AR + ID Kemenag + EN Sahih International), Sahih al-Bukhari, Sahih Muslim, Riyad as-Salihin |

## Hard rules — Sharia compliance (per PRD §12)

- **Every Islamic reference in a brief must be RETRIEVED from Qdrant, never freely generated by the LLM.** Hallucinated daleel is the single biggest credibility risk.
- Cite kitab name + verse/hadith number + page. Link back to the source passage in the UI.
- All AI outputs labelled "AI-assisted, not authoritative fatwa".
- No content that contradicts established ijma' on fundamental aqidah.
- Promote rahma (mercy) and hikmah (wisdom). No confrontational, divisive, or sectarian messaging.

## Cost discipline (per PRD §13)

- Budget cap: IDR ~1.5-2M/month (~IDR 1M LLM + ~IDR 500K VPS).
- Tiered LLM routing — Gemini Flash-Lite by default for classify/topic/rerank; Gemini 2.5 Pro reserved for brief synthesis; Claude Sonnet 4.6 only as a Pro fallback.
- Aggressive caching: never re-embed an unchanged kitab chunk; cache topic classifications.
- Batch classification calls whenever the API supports it.
- Flag any suggestion that would breach the cap before implementing.

## Local development

```bash
cp .env.example .env       # then fill API keys
docker compose up -d       # postgres + qdrant + redis
cd web && npm install && npm run dev          # http://localhost:3000
cd api && uv sync && uv run api               # http://localhost:8000
```
